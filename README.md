# pdf2ieee

A command-line tool that automatically generates IEEE-formatted BibTeX entries from PDF files. It extracts DOIs and arXiv IDs from papers, queries Crossref and the arXiv API for metadata, and outputs a `.bib` file ready to use with `IEEEtran.bst`.

## Features

- Extracts DOI or arXiv ID from PDF files automatically
- Fetches metadata from [Crossref](https://www.crossref.org/) and [arXiv](https://arxiv.org/)
- Converts IEEE journal full names to standard abbreviations (e.g., `IEEE Robotics and Automation Letters` → `{IEEE} Robot. Autom. Lett.`)
- Protects capitalization of acronyms in titles (e.g., `CNN`, `LiDAR`, `SLAM`) using `{}` guards
- Strips redundant fields (`url`, `doi`, `issn`, `publisher`) that cause unwanted `[Online]. Available:` output in IEEEtran
- Formats multi-line aligned BibTeX output
- Can fix existing `.bib` files generated by older versions

## Requirements

```
pip install requests PyPDF2
```

## Usage

**Process a folder of PDFs:**

```bash
python3 pdf2ieee.py /path/to/papers/
```

**Process specific PDF files:**

```bash
python3 pdf2ieee.py paper1.pdf paper2.pdf paper3.pdf
```

**Specify output file:**

```bash
python3 pdf2ieee.py /path/to/papers/ -o refs.bib
```

**Fix an existing `.bib` file:**

```bash
python3 pdf2ieee.py --fix-bib references.bib
```

By default, output is written to `references.bib` in the current directory.

## Output Example

```bibtex
@article{Heinzler_2020,
  title   = {{CNN}-Based {Lidar} Point Cloud {De-Noising} in Adverse Weather},
  volume  = {5},
  number  = {2},
  journal = {{IEEE} Robot. Autom. Lett.},
  author  = {Heinzler, Robin and Piewak, Florian and Schindler, Philipp and Stork, Wilhelm},
  year    = {2020},
  month   = apr,
  pages   = {2514--2521}
}

@misc{Kurup_2021_arxiv,
  title         = {{DSOR}: A Scalable Statistical Filter for Removing Falling Snow from {LiDAR} Point Clouds in Severe Winter Weather},
  author        = {A. Kurup and J. Bos},
  year          = {2021},
  month         = sep,
  eprint        = {2109.07078},
  archivePrefix = {arXiv},
  primaryClass  = {cs.CV}
}
```

## Using the Output in LaTeX

Add the following two lines before `\end{document}` in your `.tex` file:

```latex
\bibliographystyle{IEEEtran}
\bibliography{references}
```

Then compile in order:

```
pdflatex main.tex
bibtex main
pdflatex main.tex
pdflatex main.tex
```

> **Note:** The `IEEEtran.bst` style file is required. It can be downloaded from [CTAN](https://ctan.org/pkg/ieeetran) or is included in most LaTeX distributions.

## Customization

**Adding protected terms:** Edit the `PROTECTED_TERMS` list at the top of the script to add acronyms or proper nouns that should not be lowercased by IEEEtran.

```python
PROTECTED_TERMS = [
    "LiDAR", "CNN", "SLAM", ...
    "YourTerm",  # add here
]
```

**Adding journal abbreviations:** Edit the `IEEE_JOURNAL_MAP` dictionary to add journals not currently covered.

```python
IEEE_JOURNAL_MAP = {
    ...
    "Your Journal Full Name": "{Abbrev.} Journal Name",
}
```

## Limitations

- DOI and arXiv ID are extracted from the first two pages of each PDF only. Papers without either identifier must be added to the `.bib` file manually.
- Requires an internet connection to query Crossref and arXiv APIs.
- For papers that fail automatic processing, the script will print a list at the end and suggest searching [Google Scholar](https://scholar.google.com/) manually.
